# 情報設計：学生メディアアート展「stArt」Webサイト

## 要件の整理

### サイトの役割

1. 展示の雰囲気に興味を持ってもらう
2. 会期、会場、アクセスを短時間で確認できるようにする
3. 公開された作品を落ち着いて閲覧できるようにする

トップページを閲覧の中心とし、独立ページは作品一覧と作品詳細に限定する。About、開催情報、アクセス、SNSは独立ページにせず、トップページ内のセクションとして構成する。

### 優先順位

1. Heroで展示名、会期、会場、雰囲気を伝える
2. 開催情報とアクセスへ迷わず到達できるようにする
3. コンセプトによって名称と展示テーマの意味を伝える
4. 作品プレビューから一覧・詳細へ誘導する
5. SNSの開設状況と主催情報を伝える

### 固定要件

- React、TypeScript、Vite、React Routerを使用する
- GitHub Pagesのリポジトリ配下URLへGitHub Actionsでデプロイする
- HashRouterを使い、直接アクセス時の404を回避する
- UI文言は日本語にする
- 375px、768px、1440pxで表示を確認する
- 開催情報、ステートメント、SNS、ナビゲーション、作品を表示コンポーネントから分離する
- ポスター画像に文字情報を焼き込まず、HTMLで表示する
- 背景素材、ロゴ、書体、色、装飾、モーションを後から差し替えられるようにする
- 基本的なキーボード操作、フォーカス表示、aria属性、低減モーションに対応する

## 未決定事項と初期状態

| 未決定事項 | 初期状態 | 更新方法 |
| --- | --- | --- |
| 正式ロゴ | `stArt` のテキストロゴ | 設定ファイルの画像パスを指定 |
| ポスター由来の背景素材 | CSSの暗色背景、点、線、グリッド | 背景用アセットと装飾トークンを差し替え |
| 開場時間 | 項目ごと非表示 | 開催情報設定へ値を追加 |
| X、InstagramのURL | 「準備中」を表示し、リンクにしない | SNS設定へURLを追加 |
| 正式ステートメント | 支給された仮文を掲載 | 設定ファイルの文章を差し替え |
| 本番作品情報・画像 | 明示的なダミーデータとCSSプレースホルダー | ローカル作品データを更新 |
| 将来のデータ取得元 | ローカルTypeScript | Repository層をCSV、JSON、Spreadsheet向けに交換 |

## サイトマップ

- **トップ** — アプリ内パス `/`、公開URL `/#/`
  - Hero
  - コンセプト — `/#/?section=concept`
  - 開催情報 — `/#/?section=information`
  - 作品プレビュー — `/#/?section=works`
  - アクセス — `/#/?section=access`
  - SNS — `/#/?section=sns`
  - フッター／クレジット
- **作品一覧** — アプリ内パス `/works`、公開URL `/#/works`
  - 作品詳細 — アプリ内パス `/works/:slug`、公開URL `/#/works/:slug`
- **ページが見つかりません** — アプリ内パス `*`

ナビゲーションの深さは最大2段階とし、作品詳細の下へさらにページを作らない。

## ナビゲーションモデル

### 主ナビゲーション

共通ヘッダー右側へ、次の5項目をこの順で配置する。

1. コンセプト
2. 開催情報
3. 作品
4. アクセス
5. SNS

`コンセプト`、`開催情報`、`アクセス`、`SNS` はトップページ内の対応セクションへ移動する。`作品` は作品一覧ページへ移動する。

### ロゴナビゲーション

ヘッダー左側の `stArt` を選択するとトップページ先頭へ移動する。トップページですでに先頭付近にいる場合も、ページ先頭へ戻す一貫した操作とする。正式ロゴ画像が設定された場合もリンクの意味は変えない。

### 文脈ナビゲーション

- トップの作品プレビュー：`すべての作品を見る` から作品一覧へ移動
- 作品詳細：`作品一覧へ戻る`、`前の作品`、`次の作品`
- Not Found：`トップへ戻る` と、作品slug不正時は `作品一覧へ戻る`
- フッター：`ページの先頭へ`

### モバイルナビゲーション

- 主ナビゲーションをハンバーガーメニューへ変更する
- 開く操作で最初のメニュー項目へフォーカスを移す
- Escape、閉じるボタン、項目選択、ルート変更で閉じる
- 閉じた後は原則として開閉ボタンへフォーカスを戻す
- 背後コンテンツへの意図しないフォーカス移動を防ぐ
- ボタンは `aria-expanded` と `aria-controls` で状態を伝える

## HashRouterとセクション移動

HashRouterがURLのhashをルート管理に使うため、ネイティブな `#concept` はページ内アンカーに使わない。トップページ内セクションはHashRouter内のクエリ `?section=` で表す。

- 同一ページ：クエリを更新し、対応する `id` の要素へスクロールする
- 別ページ：`/?section=...` へ遷移し、トップページ描画後に対象要素へスクロールする
- 対象がない、不正な値である：ページ先頭を表示する
- 固定ヘッダー分は各セクションの `scroll-margin-top` で補正する
- `prefers-reduced-motion: reduce` の場合は即時スクロールに切り替える

許可する値は `concept | information | works | access | sns` に限定し、ナビゲーション設定と共有する。

## トップページのセクション構成

### 1. Hero

**目的**：展示名、会期、会場、雰囲気を最初の画面で理解させる。

表示順：

1. 展示名 `stArt`
2. 2026年7月29日〜31日
3. `7A101 / 7A202 / 7A207`
4. 「はじまりの星たちが、わたしたちだけの星座をつくる。」
5. 差し替え可能な背景領域

文字情報はすべてHTMLで表示し、背景素材から独立させる。初期背景はCSSによる抽象表現とする。

### 2. コンセプト

**目的**：`star`、`art`、`start` の関係と、多様な作品が展示全体で星座をつくる考えを説明する。

- 支給された仮ステートメントを意味を変えずに掲載する
- 読みやすい行長へ制限する
- 装飾的な図解は必須にせず、点と線を補助的に使う

### 3. 開催情報

**目的**：来場判断に必要な確定情報を一覧できるようにする。

- 会期
- 会場（3教室を同じ階層で並列表示）
- 開場時間（値がある場合のみ）
- 主催

ラベルと値を明確に分けた定義リストを基本とする。角丸カードの集合にはしない。

### 4. 作品プレビュー

**目的**：展示案内を読み終えた利用者へ、作品閲覧の入口を提供する。

- 公開作品の先頭から3〜4件を表示する
- 表示要素は作品画像またはCSSプレースホルダー、作品名
- 一覧では作者名を表示しない
- `すべての作品を見る` を設置する
- 公開作品が0件の場合は、グリッドを出さず準備中状態を表示する

### 5. アクセス

**目的**：移動方法と会場位置を確認できるようにする。

- `つくば駅から徒歩約10分`
- 設定されたGoogle Maps iframe
- iframeはレスポンシブな比率コンテナ内へ配置する
- iframeに会場を説明する `title` を付ける

### 6. SNS

**目的**：今後の情報発信先を伝える。

- X
- Instagram
- URL未設定時は `準備中` と表示し、リンク要素を生成しない
- URL設定後は外部リンクとして表示し、新しいタブで開く場合はその旨を支援技術にも伝える

### 7. フッター／クレジット

**目的**：主催情報とサイトの終端を示す。

- 展示名
- 主催 `GE72501 メディアアート`
- ページ先頭への導線
- 必要になった場合だけ追加クレジットを設定から表示する

## ページ別コンテンツ階層

### トップ

1. 展示名、会期、会場 — 来場判断に不可欠
2. 短いコンセプト — 興味を引く入口
3. コンセプト全文 — 展示の意味を補足
4. 詳細な開催情報 — 確定情報を整理
5. 作品プレビュー — 第二目的への入口
6. アクセス — 来場行動を支援
7. SNS、主催情報 — 継続接点と出典

### 作品一覧

1. ページ名と短い導入 — 現在地を示す
2. 公開作品一覧 — 画像と作品名を閲覧する
3. 0件時の準備中表示 — 空のグリッドを見せない
4. トップの開催情報へ戻る導線 — 第一目的へ復帰できるようにする

フィルター、検索、ページネーションは初期件数では設けない。

### 作品詳細

1. 作品一覧へ戻る導線 — 現在地と復帰先を明示
2. 作品名
3. メイン画像またはCSSプレースホルダー
4. 作者名（登録時のみ）
5. 作品説明（登録時のみ）
6. 追加画像（登録時のみ）
7. 使用技術、素材、メディウム、展示場所（項目ごとに登録時のみ）
8. 作者SNS・外部リンク（登録時のみ）
9. 前の作品／次の作品 — 公開中の作品だけを対象にする

存在しないslug、または非公開作品への一般アクセスでは作品内容を表示せず、Not Found状態から作品一覧へ案内する。

## 作品データのスキーマ

```ts
export type WorkStatus = 'draft' | 'published'

export interface WorkImage {
  src: string
  alt: string
  caption?: string
}

export interface ExternalLink {
  label: string
  url: string
}

export interface Work {
  id: string
  slug: string
  title: string
  artistName?: string
  mainImage?: WorkImage
  additionalImages?: WorkImage[]
  description?: string
  technologies?: string[]
  materials?: string[]
  media?: string[]
  exhibitionLocation?: string
  artistLinks?: ExternalLink[]
  status: WorkStatus
  sortOrder?: number
}
```

### スキーマ規則

- `id` はデータソース内で不変の識別子とする
- `slug` はURLに使用し、重複を許さない小文字英数字とハイフンを基本とする
- `title` と `status` だけを表示に必要な必須コンテンツとし、画像を含むその他の項目は任意とする
- `mainImage` がない場合は、`id` から安定して選ばれるCSSプレースホルダーを表示する
- 画像がある場合は `src` と `alt` を必須にし、装飾画像以外で空の代替テキストを許可しない
- 一覧・プレビューは `published` の作品だけを対象にする
- 前後ナビゲーションは `sortOrder`、同値または未設定時はデータ配列順で決定する
- 空配列や空文字は表示前に正規化し、空の見出しやラベルを生成しない
- 仮データの作品名・作者名は実在の人物や制作物を連想させない明示的なダミーとする

### 作品取得境界

画面は `src/data/works.ts` を直接importせず、次のRepository契約を経由する。

```ts
export interface WorksRepository {
  getPublished(): Promise<Work[]>
  getBySlug(slug: string): Promise<Work | undefined>
}
```

初期版の `localWorksRepository` がローカルTypeScript配列を読み込む。将来は同じ契約を満たすCSV、JSON、Googleスプレッドシート用実装へ交換する。データ取得中、取得失敗、0件を別状態として扱える境界にする。

### 境界の原則

- ページはデータ取得とセクション配置を担当する
- セクションは意味のあるコンテンツ単位を担当する
- 共有コンポーネントはレイアウトと操作を担当し、展示固有データを内部へ直書きしない
- 表示フォーマッタは日付、任意値、外部リンクを正規化する
- Repositoryはデータソースの違いを吸収し、画面へローカルファイル形式を漏らさない
- 視覚トークンはCSS変数に集約し、Reactコンポーネントへ色やフォントを直書きしない

## ユーザーフロー

### 来場情報を確認する

1. 利用者がトップページへ到着する
2. Heroで展示名、会期、会場、短いコンセプトを見る
3. 開催情報またはアクセスを選択する
4. 対応セクションへスクロールする
5. 開催情報とGoogle Mapsを確認する

### 展示の意味を知る

1. 利用者がHeroの短文に興味を持つ
2. コンセプトへ移動する
3. 仮ステートメント全文を読む
4. 作品プレビューまたは開催情報へ自然に読み進める

### 作品を閲覧する

1. トップの作品プレビューまたはヘッダーの `作品` を選ぶ
2. 作品一覧へ移動する
3. 画像と作品名から作品を選ぶ
4. 作品詳細を読む
5. `前の作品`、`次の作品` で連続閲覧する、または一覧へ戻る
   - 画像がない場合：CSSプレースホルダーを表示する
   - 任意情報がない場合：該当セクションを表示しない
   - 非公開または存在しない場合：作品一覧への復帰導線を示す

### 別ページから開催情報へ戻る

1. 利用者が作品一覧または作品詳細にいる
2. ヘッダーの `開催情報` を選ぶ
3. トップの `?section=information` へ遷移する
4. 描画後、固定ヘッダーを避けた位置へスクロールする

### モバイルメニューを操作する

1. 利用者がメニューボタンを選ぶ
2. メニューが開き、状態が支援技術へ通知される
3. キーボードフォーカスがメニュー内へ移る
4. 項目選択またはEscapeで閉じる
5. 選択時は目的地へ移動し、Escape時は開閉ボタンへ戻る

## コンテンツ増加への対応

- 作品は最も増加するコンテンツであり、Repositoryと公開状態フィルターで件数変化へ対応する
- 初期版は数件〜数十件を想定し、検索、分類、ページネーションは設けない
- 作品数が50件程度を超える、またはカテゴリが正式に定義された時点でフィルターとページネーションを再検討する
- 作品の並び順は `sortOrder` で運用側が管理できる
- 開催情報は単一イベントの設定として扱う。過年度アーカイブは初期スコープ外とし、必要になった時点でイベント単位のURL設計へ拡張する
- SNSとクレジットは配列で保持し、項目追加に対応する

## URL戦略

- **基本形式**：GitHub Pagesのリポジトリbase + `/#/` 以下
- **トップ**：`/#/`
- **トップ内セクション**：`/#/?section=<section-id>`
- **作品一覧**：`/#/works`
- **作品詳細**：`/#/works/<slug>`
- **動的セグメント**：作品の `slug`
- **クエリパラメータ**：初期版はトップ内移動用の `section` のみ
- **末尾スラッシュ**：アプリ内パスでは付けない方針に統一する
- **slug変更**：公開後は既存リンクを壊すため原則変更しない
- **フィルター・並び替え**：初期版ではURLへ導入しない
