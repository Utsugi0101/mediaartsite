# デザインブリーフ：学生メディアアート展「stArt」Webサイト

## 課題

会期中の来場を検討する一般来場者や学内関係者は、限られた時間の中で開催日・会場・アクセスを迷わず把握しつつ、まだ実像の見えにくい学生メディアアート展に足を運ぶ理由や雰囲気を感じたい。

現時点では正式なポスター、ロゴ、背景素材、全作品情報、開場時間、SNS URLが揃っていない。一方で、これらの未確定情報を仮デザインへ強く結び付けると、後日の更新やポスターとの統一が難しくなる。作品を過度に前面へ出すと、開催情報を探しにくくなり、実在しない学生や作品を暗示する危険もある。

## 解決方針

トップページを展示案内の中心に据え、最初の画面で展示名、会期、会場を伝える。続くセクションでコンセプト、開催情報、作品プレビュー、アクセスを一続きに閲覧できるようにし、来場判断に必要な情報へ短い経路で到達できる体験をつくる。

作品は第二の目的として、静かな一覧ページと個別ページで閲覧できるようにする。視覚表現は、深い青緑の空間、余白、文字組み、細い線、点、接続関係を用いた抽象的な星図とする。縦方向のスクロールを左右のカメラ移動へ変換し、各情報を一つの星として巡る体験をつくる。ポスター完成後は、情報構造やHTMLテキストを保ったまま、背景素材、ロゴ、フォント、色、装飾、モーションを差し替えられる構造にする。

## 対象利用者と目的

- **主要利用者**：会期中に来場を検討している一般来場者、筑波大学の学生・教職員などの学内関係者
- **主要な用事**：展示の雰囲気をつかみ、会期、会場、アクセスを確認して来場を判断する
- **第二の用事**：出展作品を一覧し、関心を持った作品の詳細を読む
- **主要言語**：日本語。展示名 `stArt`、技術名、固有名詞は必要に応じて原表記を維持する

## 成功の状態

- 初見の利用者が、最初の画面で展示名、2026年7月29日〜31日の会期、`7A101 / 7A202 / 7A207` の会場を認識できる
- トップページのナビゲーションから、コンセプト、開催情報、アクセス、SNSへ迷わず移動できる
- 作品一覧と個別ページを往復し、公開作品を前後に閲覧できる
- 未入力項目、画像のない作品、長い作品名や説明でも表示が破綻しない
- 375px、768px、1440pxで、情報階層と操作性が維持される
- ポスター完成後の視覚変更が、コンテンツや画面構造の大幅な改修を伴わない
- GitHub Pagesのリポジトリ配下URLで、直接アクセス時を含めてルーティングが機能する

## 体験原則

1. **演出の中でも案内を見失わせない** — 会期、会場、アクセスを最も読み取りやすい階層に置き、空間移動には情報の切り替わりを伝える役割を持たせる。
2. **個を並べ、関係をほのめかす** — 均一なSaaS風カードや直接的な宇宙表現を避け、余白、点、線、グリッドによって多様な作品が一つの展示をつくる感覚を示す。
3. **確定前の柔軟性を品質として扱う** — 背景、ロゴ、書体、色、コンテンツを分離し、未確定項目を無理に表示せず、後日の変更が自然に反映される設計にする。

## 美的方向性

- **哲学**：Spatial Editorial Constellation（編集的な文字組みと、移動できる抽象的な星図）
- **トーン**：知的、没入感がある、期待感がある、実験的、操作に反応する
- **参照点**：現代美術館の展覧会告知、文化施設のエディトリアルデザイン、ポスターを展開したタイポグラフィ中心のWeb表現、スイス／インターナショナル・タイポグラフィの明快なグリッド
- **反参照**：紫と青の派手なグラデーション、過剰なネオン発光、星空写真、3D惑星、ガラスモーフィズム、大量の角丸カード、SaaSランディングページ、内容と無関係なスクロールアニメーション、可読性を損なう演出
- **仮の背景**：黒一色ではなく深い青緑を基本とし、Canvasの点、細線、淡い星雲、粒子、余白によって抽象的な「個と接続」を示す。ポスター画像そのものは貼らない
- **書体**：日本語の本文・UI・見出しは `IBM Plex Sans JP`へ統一し、展示名は `Syne`。どちらもトークンから差し替え可能にする
- **モーション**：縦スクロールを連続した左右移動へ対応させ、注目中の星とコンテンツを同じ位置関係で切り替える。作品一覧・詳細へのルート遷移も進行方向に応じて左右から入れる。背景はポインター、スクロール、表示セクションへ穏やかに反応し、`prefers-reduced-motion` では静的表示へ切り替える

## コンテンツ方針

### 確定情報

- 展示名：`stArt`
- 会期：2026年7月29日〜7月31日
- 会場：`7A101 / 7A202 / 7A207`（画面上は同じ階層で並列表示）
- アクセス：つくば駅から徒歩約10分
- 主催：GE72501 メディアアート
- サイト内UI文言：日本語

### 仮掲載する情報

- コンセプト全文：支給されたステートメント案を意味を変えずに掲載する
- 作品：ダミーであることが分かる仮データを6件程度用意する
- 作品画像：具体的な作品を捏造せず、CSSによる番号、線、面の抽象的なプレースホルダーを使う

### 未確定・非表示にする情報

- 開場時間：値がない間は項目ごと非表示
- 正式ロゴ：初期版はテキストロゴ。画像が設定された場合だけ置き換える
- SNS：X、InstagramともにURLがない間はフッターにリンクを生成せず、URL設定後だけ白いアイコンを表示する
- 作品の任意項目：未入力のラベルや空の領域を表示しない

### 差し替え対象

- ポスター由来の背景画像、図形、模様
- 正式ロゴ
- 表示用・本文用フォント
- 色、罫線、装飾、モーション
- 開催情報、ステートメント、SNS URL、ナビゲーション項目
- 作品データの取得元

## 既存パターン

対象ディレクトリは空の新規プロジェクトであり、既存のアプリケーション、Gitリポジトリ、コンポーネント、CSS変数、Tailwind設定、UIフレームワーク、Storybook、デザイントークン、フォント読み込み、ページテンプレートは存在しない。

- **Typography**：新規定義
- **Colors**：新規定義
- **Spacing**：新規定義
- **Components**：すべて新規

大規模UIライブラリは導入せず、React、TypeScript、Vite、React Router、保守しやすいCSSで構築する。

## 主要な操作

- ロゴを押すとトップページ先頭へ移動する
- トップページ上のセクションリンクは対応位置へスムーズスクロールする
- 別ページ上のセクションリンクはトップページへ遷移した後、対象セクションへ正しくスクロールする
- `作品` ナビゲーションは作品一覧ページへ移動する
- 狭幅ではハンバーガーボタンでメニューを開閉し、開閉状態を `aria-expanded` で通知する
- メニュー内リンクの選択、Escape、トップページ遷移時にメニューを閉じる
- 作品を選ぶとslugを用いた個別ページへ移動する
- 個別ページから一覧へ戻る、または公開中の前後作品へ移動する
- SNS URL未設定時は操作不能なリンクや空の領域を作らない
- トップページではスクロールに合わせて各セクションが左右へ移動し、固定された注目星と接続する
- 作品一覧と個別ページは、進行方向に応じて左右から到着する
- `prefers-reduced-motion` が有効な場合は、スムーズスクロールと遷移時間を抑制する

## レスポンシブ動作

- **375px前後**：一列構成。Heroの文字サイズと余白を再構成し、ナビゲーションはハンバーガーメニューへ変更する。作品画像比率は画面幅に合わせ、地図は横幅いっぱいにする
- **768px前後**：一〜二列を内容に応じて使い分ける。作品一覧は二列を基本とし、情報欄とアクセス欄の余白を広げる
- **1440px前後**：最大幅と外側余白を保ち、Hero、開催情報、作品一覧で複数列の編集的グリッドを使う。行長を制限し、単なる拡大表示にしない
- ブレークポイントは特定端末名ではなく、コンテンツが窮屈になる位置を基準に定義する
- 作品一覧は均一な角丸カード群にせず、画像比率や余白のリズムで展示閲覧らしい静けさをつくる

## アクセシビリティ要件

- 本文テキストは通常文字でWCAG AA相当の4.5:1以上、大きな文字で3:1以上のコントラストを確保する
- すべてのリンクとボタンをキーボードで操作可能にし、暗い背景上でも明確な `:focus-visible` を表示する
- モバイルメニューに適切なラベル、`aria-expanded`、`aria-controls` を付け、開閉後のフォーカスを管理する
- 見出し順、ランドマーク、リスト構造を意味に沿って使用する
- 作品画像に内容に応じた代替テキストを設定し、装飾要素は支援技術から隠す
- Google Maps iframeに会場を説明する `title` を付ける
- 現在地や遷移結果をページタイトルとフォーカス移動で伝える
- 色だけで公開状態や操作状態を伝えない
- タップ対象は十分な大きさと間隔を確保する
- `prefers-reduced-motion` を尊重する

## 技術・運用上の制約

- React、TypeScript、Vite、React Routerを使用する
- GitHub Pagesのリポジトリ配下URLへGitHub Actionsで自動デプロイする
- Viteの `base` はActions上でリポジトリ名に追従させ、ローカルでは `/` を使用する
- 初期ルーティングはHashRouterを用いる
- 開催情報、ナビゲーション、SNS、ステートメントは一か所の設定ファイルで管理する
- 作品データは画面から分離し、初期版はローカルTypeScriptデータと交換可能な取得層を使う
- 背景素材とロゴはパスやCSS変数の変更で差し替え可能にする
- タイトル、会期、会場などの文字情報は画像へ焼き込まない
- マップは支給されたGoogle Maps埋め込みURLを設定ファイルから読み込む
- READMEに運用、デプロイ、コンテンツ更新、将来のCSV／Googleスプレッドシート移行方法を記載する
